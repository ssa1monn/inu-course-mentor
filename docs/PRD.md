## 1. 개요 (Overview)

**INU Course Mentor**는 인천대학교 학생을 위한 **개인 맞춤 수강·학사 조언 웹 서비스**다.

학교가 배포하는 **학부 종합강의시간표 엑셀(.xlsx)** 을 업로드하면 그 학기 전체 강의를
데이터베이스화하고, 학생이 입력한 프로필(학과·학년·학기·관심 직무·수강 이력)을 바탕으로
**졸업요건 충족 현황을 자동 계산**한다. 여기에 **생성형 AI(Gemini)** 와(가능하면) 외부 진로
데이터를 결합해 "무슨 과목을 왜 들어야 하는지"를 자연어로 안내한다.

한 줄 정의:
> "엑셀 한 장 → 내 졸업까지 남은 길 + 이번 학기 뭘 들어야 하는지를 알려주는 AI 학사 멘토"

---

## 2. 문제 정의 (Problem)

- 인천대 종합시간표는 **2,400여 개 강의가 든 거대한 엑셀 한 장**이라 학생이 직접 보기 어렵다.
- 졸업요건(전공필수/전공심화/기초교양/총학점 등) 충족 여부를 학생이 손으로 계산해야 한다.
- "내 진로에 맞는 과목이 뭔지", "이번 학기 우선순위가 뭔지"를 알려주는 도구가 없다.
- 시간표를 짤 때 **강의 시간 충돌**을 일일이 눈으로 확인해야 한다.

INUCM은 이 네 가지 불편을 하나의 흐름으로 해결한다.

---

## 3. 대상 사용자 (Target Users)

| 페르소나 | 상황 | INUCM으로 얻는 것 |
|---|---|---|
| **재학생 (주 사용자)** | 매 학기 수강신청 전 시간표를 고민 | 졸업요건 현황 + 맞춤 과목 추천 + 충돌 없는 가상 시간표 |
| **저학년 신입생** | 무슨 과목부터 들어야 할지 모름 | 학년/학기별 수강 로드맵, 선수과목 안내 |
| **졸업 임박 학생** | 졸업까지 부족한 영역 파악 필요 | 영역별 부족 학점 + 이번 학기 보완 제안 |

데모/평가자(교수님)도 회원가입 → 엑셀 업로드 → 추천까지 7분 영상 안에서 볼 수 있어야 한다.

## 4. 목표와 성공 기준 (Goals & Success Criteria)

### 제품 목표
1. 엑셀 업로드 → 강의 DB 적재까지 **한 번의 클릭**으로 동작.
2. 프로필 입력 → 졸업요건 충족률을 **영역별로 시각화**.
3. 강의 검색·필터 + 가상 시간표 + **시간 충돌 자동 감지**.
4. 위 정보를 묶어 AI가 **읽을 만한 자연어 조언** 생성.

### 과제 평가 기준 (프로젝트_안내.pdf — 필수 충족)
- [ ] **CI/CD**: GitHub Actions 등으로 자동 배포 환경 구성
- [ ] **Docker, Nginx** 등 수업 기술 요소 적극 활용
- [ ] **Web Storage + DBMS** 둘 다 활용
- [ ] **배포된 접속 가능 URL** 제출
- [ ] **7분 이내 발표 영상** (서비스 소개 + 주요 기능 시연)

### 성공의 정의(데모 시나리오 통과)
> 신규 사용자가 가입 → 프로필 입력 → 종합시간표 엑셀 업로드 → 강의 목록 확인 →
> 졸업요건 충족률 확인 → AI 조언 받기 → 가상 시간표 구성(충돌 감지) 까지
> 막힘 없이 진행되고, 새로고침/재로그인 후에도 데이터가 유지된다.

## 5. 핵심 기능 (Core Features)

설계 문서의 6대 기능을 사용자 스토리로 구체화한다. 우선순위: **P0**(MVP 필수) > **P1**(중요) > **P2**(여유 시).

### F1. 회원가입 · 프로필 (P0)
- 사용자는 이메일/비밀번호로 가입·로그인한다. (JWT 인증)
- 가입 시 또는 마이페이지에서 프로필을 입력한다:
  - 학과, 입학년도, 현재 학년, 현재 학기
  - 관심 직무 (예: 백엔드 개발, 데이터 분석 …)
  - 과목 선호도 (이론/실습 비중, 관심 분야)
- 프로필은 졸업요건 계산과 AI 추천의 기준 데이터가 된다.

### F2. 종합시간표 엑셀 업로드 · 파싱 (P0)
- 사용자가 `.xlsx` 종합시간표를 업로드한다.
- 백엔드가 파싱하여 강의 데이터를 DB에 적재한다:
  학수번호, 교과목명, 담당교수, 학과, 이수구분, 학점, 강의실, 요일/시간.
- 같은 학기를 재업로드하면 기존 데이터를 갱신(upsert)한다.
- RISE 등 **시간표 없는 강의**도 깨지지 않고 적재된다.

### F3. 강의 조회 · 검색 (P0)
- 학과 / 이수구분 / 요일 / 시간대 / 키워드로 강의를 검색·필터링한다.
- 강의 카드에 핵심 정보(과목명/교수/학점/시간/강의실)를 표시한다.

### F4. 가상 시간표 · 충돌 감지 (P1)
- 관심 강의를 "담기" 하여 가상 시간표를 구성한다.
- 담는 순간 기존 강의와 **시간 충돌**을 감지해 경고한다.
- 구성 중인 시간표는 브라우저(Web Storage)에 즉시 저장되어 새로고침에도 유지된다.

### F5. 졸업요건 충족 자동 계산 (P1)
- 학생의 수강 이력 + 가상 시간표를 졸업요건 규칙과 대조한다.
- 영역별(전공필수/전공심화/기초교양/총학점 등) **충족률과 부족 학점**을 산출·시각화한다.

### F6. AI 맞춤 조언 (P1)
- 백엔드가 (프로필 + 졸업요건 요약 + 후보 강의)를 컨텍스트로 묶어 **서버에서** AI에 전달.
- AI는 자연어 조언을 생성한다:
  - 희망 직무에 맞춘 추천 과목과 이유
  - 현재 학년/학기 우선 수강 로드맵
  - 졸업요건 중 부족 영역 보완 제안
  - 선수과목 리마인드
- 생성된 조언은 학기별로 저장되어 재조회 가능하다.

### F7. 개인 이력 저장·관리 (P2)
- 추천/조언/시간표/로드맵을 학기별로 저장하고 다시 조회한다.

## 6. 범위 밖 (Out of Scope)

- 실제 인천대 수강신청 시스템과의 연동(로그인/신청 대행) — 하지 않는다.
- 모바일 네이티브 앱 — 반응형 웹으로만 대응.
- 다국어 — 한국어 우선.
- 결제/유료 기능 — 없음.
- 학생 간 소셜/공유 기능 — MVP 범위 밖(여유 시 P2).

## 7. 제약 · 가정 · 리스크 (Constraints / Assumptions / Risks)

### 제약
- **시간**: 시작 시점 기준 마감까지 약 2일. 범위 통제가 최우선.
- **필수 기술**: Docker, Nginx, CI/CD, Web Storage, DBMS 반드시 사용.
- **보안**: API 키 등 민감정보는 `.env`로 관리, 클라이언트/이미지에 노출 금지.
- **로그/주석에 이모지 사용 금지** (프로젝트 코드 스타일 규칙).

### 가정
- 업로드 엑셀은 학교 표준 포맷(20컬럼, `sheet1`, 1행 제목·2행 헤더·3행~데이터)을 따른다.
  → 상세는 `SPEC.md`의 엑셀 파싱 규격 참고.
- 졸업요건 규칙은 데이터로 관리하며, 초기엔 컴퓨터공학부 기준 샘플 규칙으로 시작한다.
- AI는 Gemini 무료 티어를 사용한다.

### 리스크 & 대응
| 리스크 | 영향 | 대응 |
|---|---|---|
| 2일 내 풀스택+배포+영상 완수 어려움 | 높음 | MVP(P0) 먼저 완성 후 P1, P2 순차 추가 |
| 커리어넷 API 키 미발급 | 중간 | 외부 진로 API는 P2/선택. AI 단독 조언으로 폴백 |
| 엑셀 시간 포맷 예외(다중요일/null) | 중간 | 파서에 예외 처리 + null 시간표 허용 (SPEC 참고) |
| 졸업요건 규칙의 정확성 | 중간 | "데모용 샘플 규칙"임을 UI에 명시, 구조는 확장 가능하게 |
| Docker 배포 시간 소요 | 중간 | 로컬 docker-compose 우선 검증 후 배포 |
